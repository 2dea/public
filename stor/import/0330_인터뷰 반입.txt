열쇠관리
	1. 예비열쇠관리
- 위탁열쇠 대상은 무조건 6개(금고 2개 사용 시 예외적으로 위탁 수량 늘어날 수 있음)
- 현금금고 2개 사용 중이나 현금을 보관하지 않을 시 위탁하지 않아도 된다고 안내하고 있음
- 현금금고일 경우, 현금보유 여부 체크 후 현금보유 중인 것만 위탁 예비열쇠로 포함
- 위탁대상 예비열쇠만 위탁내역 그리드에 표시(위탁구분이 위탁인 경우만 표시)
- 체크박스 선택 후 일괄처리 필요한 버튼만 상단에 위탁요청, 반납요청
- 각 위탁가능 예비열쇠 내역별 처리 가능한 버튼 컬럼에 표시 
- 상단(우리지점) : 우리지점 위탁가능 열쇠내역(우리지점이 보유한 위탁가능 열쇠 노출)
> 버튼 : 위탁요청, 위탁취소, 반납요청, 반납요청취소
- 하단(상대지점) : 우리지점에서 받은 열쇠만 노출
> 버튼 :  위탁승인, 반납승인
- 열쇠를 위탁하거나, 반납요청 할때도 열쇠주인만 가능하도록 기능 구현
- 위탁안된 열쇠는 빨간색으로 표시해서 위탁필요 표시 
- 그리드 하단에 위탁구분이 위탁인 내역만 표기됩니다. 안내문구 노출
- 메인에 위탁가능 열쇠 건수, 위탁 중 열쇠 건수(미처리 조치방법 가이드 필요)
- 반납이 완료되면 반납내역 그리드에서 사라짐
- 열쇠등록관리 상태 값 변경 : 위탁미완료 시 위탁(미지정), 위탁완료 시 위탁(을지로 6가)
- 반납열쇠 재위탁 가능하도록 반납사유(임시반납) 
- 재위탁 시 안내문구 제공 (열쇠수정이 아닌 신규 등록 필요)
> 기존 열쇠내역에서 수정하려는 경우 얼럿 노출
    반납이력이 존재합니다. 새로운 열쇠는 신규 등록하셔야 합니다.(확인)
- 반납은 무조건 폐기이나 집에 잠깐 두고 오는 경우의 반납은 이력만 남김(문의 시 반납 받으시고 재위탁하라고 안내)
	2. 열쇠등록관리
- 관리책임자변경, 사용열쇠인수도 버튼 삭제
- 거래내역, 인수도내역, 예비열쇠위탁내역 탭 삭제(관련조회 및 출력화면에서 확인)
- 인수도내역 탭 > 열쇠인수도내역 화면으로 이동
- 등록관리화면 내 자동화기기 ATM선택 시 비고입력 필수
- 단말번호 > ATM단말번호로 변경
	3. 열쇠인수도내역
- 화면 좌/우 분할하여 좌측에 목록, 우측에 인수도내역
- 컬럼 삭제 : 관리책임자여부, 상태, 금고실세트카드, 열쇠상태, 열쇠종류, 비고
- 컬럼남김 : 결재상태, 보관책임자, 이전보관책임자, 취소일자, 취소사유
- 단말번호 : ATM단말번호
	- 열쇠인수도 : 예비열쇠보관, 관리책임자 컬럼 삭제
	4. 관련조회 및 출력
-  예비열쇠위탁내역 화면에 예비열쇠보관내역 화면 통합
	5. 예비열쇠발송요청
- 매달 보관증 발송 필요(위탁내역 존재 시 일괄 알림 발송)
- 알림요청은 직원선택할 수 있도록 제공, 확인증 팀장결재
- 팀장 본인이 결재 시 팀장 자동승인(기안자가 팀장일 경우)
- 예비발송확인증 수령 탭 내 알림버튼 추가
- 예비열쇠보관확인증발송요청알림 (직원선택, 자동메시지) UC알림

보고서 관리
	1. 자위소방대편성표 안전기획팀으로 이관
	2. 비상연락망 기능 제외
	3. 연간자율보안체계 > 자율보안체계 메뉴명 변경
	4. 진척률 삭제(개별 업무 시기 상이)

캘린더 신청 프로세스 관련 
	1. 캘린더 신청기간
- 1차 정기신청, 2차 배송신청 5월에 일괄신청
	2. 신청기간 경과 후 영업점 신청 건에 대해서 기간조정 가능하도록 브랜드전략실에 권한 부여(AS-IS 동일)
	3. 5월 캘린더 신청기간에 영업점별 배부수량에서 1차 정기신청 건, 2차 배송신청 건 나눠서 일괄신청 받은 후 
11월 배송 전에 재확인 > 재확인 시 2차 배송신청 필요 없어질 경우 영업점 수량으로 포함 
	4. 정기신청(5월), 배송신청(11월)
	5. 1차 신청 시 캘린더 A, B, C 품목별 총 수량만 신청 > 2차 신청 시 품목별 신청수량 내에서 업체(고객) 배송 수량 
예) 배부수량 1,000개 
① 1차 신청기간에 영업점에서 A, B, C 품목 총 800부 신청 
- A : 300, B: 300, C: 200
② 2차 신청기간에 A, B, C 폼목 신청수량 내에서 고객 배송정보 등록 
- 각 품목별 고객정보 등록 시 총 수량에서 마이너스 처리 
- 고객정보 등록 후 남은 수량은 영업점으로 배송 
	6. 캘린더 배송진행현황 미표시, 대신 각 품목별 제작업체 담당자 이름, 전화번호 표시 필요(매년 입찰로 업체 선정)
- 제작업체정보 등록 및 수정 기능 제공
- 캘린더 배송이후 절차 본부에서 관여하지 않음
	7. 고객정보 엑셀업로드, 화면 내 주소입력 시 주소검색 없이 텍스트 입력
- 수취인명(고객명/업체명),  주소, 전화번호
	8. 서무권한(대리, 팀장 등)을 가진 직원은 누구나 신청 가능
	9. 1차, 2차 신청 부점장 결재
	10. 알림발송 기능 제공
- 시행문 등록 후 수동으로 알림발송(UC메신저)
- 업무포털 내 나의 할 일에 미처리 건수 표시, 클릭 시 업무지원시스템 메인으로 링크(메인화면 내 캘린더 미신청 건 표시) 
- 부점장 미결재 시 미처리 건수로 계속 표시(서무권한자 전체) 

화면설계 수정
	1. 캘린더신청 영역 공지사항 영역과 위치 교체
- 좌측에 공지사항, 우측에 캘린더신청
	2. 공지사항 이미지 관리 기능 제공
- 영업점 유형별 노출되는 공지사항 상이
	3.  신청수량 (+) 클릭 시 25개(박스단위) 단위로 상향 
	4. 결재요청, 수정결재요청 버튼명 수정
- 결재요청 전 수정결재요청 버튼 비활성화, 결재요청완료 시 수정결재요청 버튼 활성화
	5. 인수등록 > 인수증출력으로 버튼명 변경
- 인수증출력 버튼 하나로 통일 
- 인수증 버튼 클릭 시 폼목 리스트 전체 노출, 체크박스 선택 후 인수증 출력
- 인수증 출력화면에 체크된 리스트 전체 노출
	6. 기능 추가
-  1차 신청 화면 내 공지사항 이미지 관리, 달력 카테고리 관리 
> 달력 카테고리 관리 화면 내 캘린더 제작업체 정보, 박스단위 변경 기능 제공
- 2차 신청 화면내 캘린더 제작업체 관리(업체명, 담당자, 연락처), 고객주소 관리 팝업 추가
