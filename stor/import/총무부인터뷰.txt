용도품 계정관리
	1. 용도품계정출금내역조회 
- 통합자산에서 용도품신청내역 검수 후 등록되는 것인지 생성시점 알 수 없음
- 총무부에서는 조회 및 사용하지 않음(조회 데이터는 필요)
- 태규영대리님께 확인 필요
	2. 용도품계정대사
- 총무부에서 사용하지 않으나 데이터는 생성되고 있음
- 계정대사를 하기위해 출금내역이 필요한 것인지 확인 필요
- 업체직배송품목정산에서 단가 입력 후 > 용도품, 통장인지세 납부를 이슬아 과장님께 작업요청 → 대금정산 > 대금정산품의 메뉴에 정보 노출
 ㄴ 부점별 데이터를 파일로 받아서 매달 IT쪽에 인지세 정보 전달

대금정산(용도품, 인지세)
	1. 인지세는 매달 처리, 용도품은 업체에 대금정산은 매달 지급하나 CC처리 3개월에 한번으로 영업점에서 출금(매달 처리하지 않는 이유 알 수 없음)
	2. 업체직배송품목 정산
영업점에서 A4용지 신청하면 확인 후 
업체직배송으로 영업점에서 받았는지 프로세스혁신부에서 확인불가, 3개월 지난 시점에 재신청할 경우 인수하지 않은 내역이 있습니다. 안내메시지 노출되고 있음(인수확인은 영업점에서 불가)
- 인수확인 하지 않은 상태에서 대금정산되고 있음
       배송 후 2주 시점에 미인수내역에 포함하는 의견 제시 - 인수확인처리 영업점에서 가능하도록
       - 프로세스 : 프혁에서 확정 > 대금정산품목정산에 내역 노출 > 3개월에 한번 생성
       - 3개월에 자료 생성(프혁에서 사전 작업완료)> 단가는 일괄등록 가능(통합자산에서는 업체별 품목 단가 입력해 놓고 자동 세팅)
      - 통합자산(서울시, 서울시 외, 본점)에서 계약관리(업체/단가) > 통합자산에서 단가 불러오는 방향으로 정의 (계약된 날짜 지나면 단가입력부분 공간으로 표시되고 있음)
- 불편함은 없음
- 용도품 CC하는 과정에 A4용지 금액도 합산되어 처리 
	3. 기타대금정산 사용하지 않음 - 피복규정 변경(자율화)되어 메뉴 삭제
	- 가디건은 은행 총자산에서 비용처리
	4. 정산내역조회
- 이슬아 과장님께서 작업한 내용이 노출되고 있음 - 엑셀작업 전산화 여부 이슬아 과장님께 확인 필요(총무부에서 다운받을 수 있도록 메뉴 유지)
	5. 대금정산품의
- 이슬아 과장님께서 작업한 내용 노출되고 있음 - 엑셀다운 받아서 전자결재하고 있음 (전자결재 가능하도록 양식 전산화하는 방향 - 부장님 결재 후 CC처리하는 담당자에서 재결재 필요)
- 결재요청 버튼 삭제 필요
- 예산작액체크 - 총무부에서 보유하고 있는 예산 체크
	6. 검수계정처리조회
- 태규영대리님께서 신청 후 검수완료된 내역 노출(태규영 대리님께 확인 필요)
- 총부부에서는 사용하지 않음


서식관리(총무부/프혁부만 사용)
	1. 출급번호 관리 - BPR, 중요용지/용도품(통장류 제외) 출급번호 체번 (1. 서식코드 > 2. 출급번호)
- 출급번호 채번 자동으로 안되고 있음(분류별로 앞자리(2~3자리)는 세팅되고 있음)
- 채번 가장 마지막에 채번된 숫자 다음으로 자동 채번
- 양서식을 경우 서식코드 먼저 채번 후 출급번호
- 서식 외(카드) 경우 출급번호만 채번(서식코드 필요없음)  
- 일반양서식이어도 출급번호 필요한 부분 존재
- 영업점에서 자체 양식제작 시 총무부에 요청 - 서식코드 채번해줌(영업점에서 용도를 총무부로 전자결재 옮림)
	2. 서식코드관리
- 첨부파일 출급번호와 동일하게 등록/변경 시 추가
- 양서식에 동시 등록(선택상황)
	3. 서식- 출급- BPR 순으로 채번
- 양서식 BPR 사용여부 체크 시 출급번호 생성됨
